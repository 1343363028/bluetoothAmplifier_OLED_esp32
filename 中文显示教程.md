## 1. 硬件准备

| 元件 | 型号 | 接线 |
|------|------|------|
| 主控 | ESP32 Dev Module | — |
| 显示屏 | SSD1306 OLED 128×64 | I2C：SCK→GP22, SDA→GP21 |
| 电源 | 5V | VDD→5V, GND→GND |

> 其他 OLED（如 SH1106、128×32）也适用，只需修改构造函数参数。

---

## 2. 为什么中文显示这么麻烦？

ASCII 字符只有 128 个，用一个字节就能编码，字体数据也小（通常 5×7 像素）。所以 `Adafruit_SSD1306` 这类库默认就支持，直接 `display.print("Hello")` 就能显示。

中文不一样：

- **字符数量大**：常用汉字就有几千个，全量 GB2312 字符集 6763 个字
- **编码是多字节**：UTF-8 下一个汉字占 3 字节，GBK 下占 2 字节
- **字体数据大**：12 像素全量中文字体约 500KB，远超 ESP32 默认程序分区

所以需要：换支持中文的库（U8g2）→ 选合适字体 → 改分区方案 → 用对 API。下面逐步讲解。

---

## 3. 第一步：安装 U8g2 库

1. 打开 Arduino IDE
2. 菜单栏：**工具 → 管理库**
3. 搜索 `U8g2`
4. 作者选 `oliver (olikraus)`，点安装

> 不要用 `Adafruit_SSD1306`，它没有内置中文字体，自己制作字模很麻烦。

---

## 4. 第二步：选择中文字体

U8g2 内置了大量中文字体，常用的有：

| 字体名 | 大小 | 说明 |
|--------|------|------|
| `u8g2_font_wqy12_t_gb2312` | ~500KB | 文泉驿 12px，覆盖全 GB2312，**推荐** |
| `u8g2_font_wqy13_t_gb2312` | ~500KB | 文泉驿 13px |
| `u8g2_font_wqy15_t_gb2312` | ~600KB | 文泉驿 15px |
| `u8g2_font_ncenB08_tr` | 小 | 仅 ASCII，**不支持中文** |

> 字体名末尾的 `_t` 表示 transparent（透明背景），`_gb2312` 表示字符集。初学者无脑用 `u8g2_font_wqy12_t_gb2312` 即可。

---

## 5. 第三步：最小可运行示例

新建一个 `.ino` 文件，粘贴以下代码：

```cpp
#include <U8g2lib.h>
#include <Wire.h>

// 构造函数：SSD1306 128×64, I2C
// U8G2_R0 表示不旋转，U8X8_PIN_NONE 表示用默认 I2C 引脚
U8G2_SSD1306_128X64_NONAME_F_HW_I2C u8g2(U8G2_R0, /* reset=*/U8X8_PIN_NONE);

void setup() {
  u8g2.begin();
  u8g2.setFont(u8g2_font_wqy12_t_gb2312);   // 设置中文字体
  u8g2.enableUTF8Print();                    // 启用 UTF8 打印（可选）
}

void loop() {
  u8g2.clearBuffer();

  // ★ 关键：用 drawUTF8 而不是 print
  u8g2.drawUTF8(0, 12, "你好，世界！");
  u8g2.drawUTF8(0, 30, "Hello, ESP32");

  u8g2.sendBuffer();
  delay(1000);
}
```

接线：SDA→GP21，SCK→GP22，VDD→5V，GND→GND。

上传到 ESP32，如果一切正常，OLED 上会显示两行文字。

## 6. 第四步：解决 Flash 空间不足

上传时如果报错：

```
Sketch too big; see https://support.arduino.cc/hc/en-us/articles/360013825179
```

原因是默认分区方案给程序的 Flash 空间只有 1.3MB，中文字体 + 蓝牙协议栈编译后约 1.4MB，超出限制。

**解决方法**：

1. 菜单栏：**工具 → 分区方案**
2. 选择 **"Huge APP (3MB No OTA/1MB SPIFFS)"**

这会把程序空间扩到 3MB，代价是放弃 OTA 无线升级。课堂项目一般用不到 OTA，可以放心选。

> 重新编译后，IDE 底部状态栏会显示占用百分比，应降到 47% 左右。

## 7. 第五步：解决中文乱码

如果 OLED 上显示的是乱码（方块、问号、错位字符），按以下顺序排查：

### 7.1 检查 API 用法

**最常见的错误**：用 `print()` 显示中文。

```cpp
// ❌ 错误：print 把 UTF-8 中文按单字节送显，必然乱码
u8g2.setCursor(0, 12);
u8g2.print("你好");

// ✅ 正确：drawUTF8 识别 UTF-8 多字节序列
u8g2.drawUTF8(0, 12, "你好");
```

**原因**：
- `print()` 是 Arduino 通用接口，逐字节输出字符。UTF-8 下一个汉字占 3 字节，被当成 3 个独立字符显示
- `drawUTF8()` 是 U8g2 专门为 UTF-8 设计的函数，正确解码多字节序列后查表显示

### 7.2 检查源文件编码

`.ino` 文件必须保存为 **UTF-8** 编码。

Arduino IDE 默认就是 UTF-8，一般不用管。如果用 VSCode 编辑，检查右下角编码标识，不是 UTF-8 就点一下转码。

### 7.3 检查字体是否包含所需字符

`u8g2_font_wqy12_t_gb2312` 覆盖 GB2312 字符集（6763 字），生僻字可能不在其中。换成 `u8g2_font_unifont_t_chinese2` 覆盖更广。

## 8. 第六步：动态布局（进阶）

固定坐标写死文字位置有个问题：不同字数下文字会重叠或错位。

**例子**：音量条想紧跟"音量："两字之后，但 12px 字体下"音量："实际占 36px，写死坐标就会重叠。

**解决**：用 `getUTF8Width()` 动态测量文字宽度。

```cpp
const char* label = "音量：";
int labelWidth = u8g2.getUTF8Width(label);   // 动态获取像素宽度
u8g2.drawUTF8(0, 60, label);
u8g2.drawBox(labelWidth, 50, volume * 2, 8);  // 进度条紧跟文字
```

这样无论字体大小或文字内容怎么变，布局都不会乱。

## 9. 常见问题排查

| 现象 | 原因 | 解决 |
|------|------|------|
| `Sketch too big` | 默认分区 1.3MB 装不下中文字体 | 改分区为 Huge APP |
| 中文显示为方块/乱码 | 用了 `print()` 而非 `drawUTF8()` | 改用 `drawUTF8()` |
| 部分生僻字不显示 | 字体字符集不全 | 换 `unifont_t_chinese2` |
| 编译报错找不到字体名 | 字体名拼写错误 | 复制官方文档字体名 |
| OLED 全黑不亮 | I2C 地址不对 | 试着改构造函数为 `..._2nd_hw_i2c` 或扫描 I2C 地址 |
| 文字方向反了 | 屏幕旋转方向 | 构造函数参数改 `U8G2_R1`/`R2`/`R3` |

### I2C 地址扫描脚本

如果 OLED 不亮，先用这个脚本确认 I2C 地址：

```cpp
#include <Wire.h>
void setup() {
  Serial.begin(115200);
  Wire.begin();
  for (uint8_t addr = 1; addr < 127; addr++) {
    Wire.beginTransmission(addr);
    if (Wire.endTransmission() == 0) {
      Serial.print("Found I2C device: 0x");
      Serial.println(addr, HEX);
    }
  }
}
void loop() {}
```

SSD1306 通常是 `0x3C` 或 `0x3D`。

## 10. 总结：四要素检查清单

中文显示不正常时，按这个清单逐项检查：

- [ ] **库**：用 U8g2，不是 Adafruit_SSD1306
- [ ] **字体**：`u8g2_font_wqy12_t_gb2312` 或其他带 `_gb2312`/`_chinese` 后缀的字体
- [ ] **分区**：Arduino IDE 选 "Huge APP (3MB No OTA/1MB SPIFFS)"
- [ ] **API**：用 `drawUTF8()`，不是 `print()`

四项全部满足，中文就能正常显示。其中 **API 用错**（`print` vs `drawUTF8`）是初学者最容易踩的坑，因为编译不报错，运行才乱码。
